# Framework：局势分析框架

给 agent 在每次输出前做局势判断用的结构化工具。**先画图、再选招**。

---

## 1. 权力三角图

任何一场职场博弈都至少有三方。把对方的"话"还原成"谁想让谁做什么"。

```
          BigBoss / Skip
                 ▲
                 │ (升级路径，我方的终极筹码)
                 │
               Boss
              ╱     ╲
   (指令)  ╱         ╲  (授权 / 授意)
          ▼            ▼
         Me   ◀────▶   QA_Gatekeeper / Peer
               (表面博弈)
```

**关键问题（每次都要答）：**

1. **谁是决策人？** 表面跟我说话的人往往不是。QA_Gatekeeper 的底气来自 Boss 的授意。
2. **谁是白手套？** Boss 不亲自下场，让 QA_Gatekeeper / HR / 其他 leader 挡在前面。
3. **谁在旁观？** 同级、群里其他人、老板的老板。旁观者是未来的盟友 / 证人 / 举报对象。
4. **我能越级吗？** 如果能，我就有筹码；如果不能，必须先建越级通道。

---

## 2. 叙事框架（对方 VS 我方）

对方把议题塞进**他有利的框架**里，我就输了一半。每次必须识别框架、再选择**接框 / 换框**。

| 框架 | 对方用法（典型例子） | 对我方不利点 | 换框策略 |
|------|---------------------|------------|---------|
| **成本框架** | "对你们来说成本就是一次转让人" | 把我方真实成本说没了 | 拆分成本：时间、响应、心智、累计 |
| **中心点 / 单点风险** | "<QA_Gatekeeper> 会是中心点，休假就没人处理" | 把正常的职责说成风险 | 指出对方本身就是中心点 + 要求建 backup |
| **人力借口** | "我们没有人力支持这个" | 模糊数字，不可验证 | 要求量化：几个人日？谁排期？ |
| **权威框架** | "leader 没改就是 old 的" | 把对方老板拉来压人 | 要求出处：哪位 leader？文档链接？ |
| **打人设** | "你看起来情绪很大" | 把讨论变成情绪管理 | 把话题拽回具体事项 + 要求书面化 |
| **机器人 / 工具背锅** | "机器人分拣要保证准确" | 把人的责任推给系统 | 问：谁是系统 owner？准确率数据？ |
| **AI / 通用方案** | "<AI 战略代号> 不是说要支持 ai 能力吗" | 用大词盖过具体问题 | 要求给出具体产品 / 具体 owner / 具体排期 |

**换框三步走：**

1. **不接对方的词**（不要在回复里重复他的框架词，否则你就在他的叙事里讨论）
2. **把议题重新定义**（"这不是成本问题，是责任边界问题"）
3. **把举证责任丢回去**（"按你说的 X，那 X 的数据 / owner / 排期是？"）

## 3. 书面化原则

> **口头的话都不算数。没写进文档 / 群 / 邮件的承诺，等于没承诺。**

所有进攻型招式最终都要收敛到书面化。书面化的层级：

| 层级 | 渠道 | 适用 |
|------|------|------|
| L1 | 私聊 IM | 最弱，对方随时翻脸不认 |
| L2 | 工作群 IM | 有旁观者，基本能留痕 |
| L3 | Confluence / 飞书文档 / Jira 评论 | 正式，可被搜索 |
| L4 | 邮件 + 抄送相关方 | 非常正式，不可抵赖 |
| L5 | 评审会议纪要 + 签字 / 回复确认 | 最强，可用于 HR / 升级 |

**每一轮对话的目标至少把局面推高一级。**

## 4. 风险评估（选档位的依据）

选**温和 / 标准 / 强硬**前，快速打一下这张表：

| 维度 | 低风险 | 中风险 | 高风险 |
|------|--------|--------|--------|
| 我离职成本 | 手里有 offer / 能走 | 近期不想走 | 被绑死（签字费 / 期权 / 签证） |
| 老板报复能力 | 有大老板庇护 / 横向可调岗 | 一般 | 绝对权威，能让我 PIP |
| 事件可公开度 | 有群、有邮件、多人在场 | 半公开 | 纯私聊，对方能删 / 否认 |
| 我方数据 / 证据 | 完整链路可截图 | 部分有 | 只有自己的记忆 |
| 盟友数量 | 有同级 / 上级支持 | 中立 | 孤立 |
| 业务重要性 | 我业务是老板 KPI | 一般 | 边缘业务 |

**简单打分规则：**

- 6 项全低风险 → 可以走 **强硬档**
- 多数中风险 → 走 **标准档**
- 任意一项高风险（尤其离职成本高 + 孤立）→ 走 **温和档 + 私下积累证据**，先把权力图变好看再说

## 5. 诉求拆解（我到底要什么）

用户常把"我想怼他"当成诉求，但真实诉求通常是下列之一：

- **责任边界**：不让这个锅进我这边
- **权力边界**：不让对方插手我们的流程
- **信息对称**：让大老板知道对方在搞小动作
- **关系止损**：不撕破脸但也别再被欺负
- **情绪发泄**：单纯想出口气（skill 要识别出这种，**不要**给用户发泄型话术，改成"写个草稿但建议别发"）

**agent 必须在输出前明确本轮的真实诉求**，不然三档方案就会散。

## 6. 常见局面速判表

| 症状 | 局面诊断 | 首选策略方向 |
|------|---------|------------|
| 老板让我去说服 QA / 产品 / 其他团队，对方总不配合 | 老板把协调成本甩给我，自己不做决策 | 把决策权问题正式提出来：要么老板拍板，要么让我有对等权力 |
| QA 推三阻四 + 老板帮 QA 说话 | QA 是白手套 | 不跟白手套缠斗，直接把议题推到 Boss 或 BigBoss |
| 被指"没有 AI 思维 / 不够创新" | 模糊化 PUA，等你自证 | 反问：具体是哪件事？期望动作是？让对方给具体例子 |
| 会议上被点名甩锅 | 公开埋点攻击 | 会议纪要书面化 + 会后追一句"刚刚会上你说的 X，我理解是 Y，对吗？" |
| 周报 / OKR 被暗戳戳打低 | 准备秋后算账 | 立即要求 1on1 对齐口径 + 把反馈落成书面文档 |
| 被塞活 + 又被说"这不是你该管的" | 边界钓鱼，试探你的底线 | 要求一次性明确 RACI，书面化 |

## 7. 沉默成本与时间维度

> **对方越急你越稳。你越急对方越赚。**

- 对方 @ 你要求立刻回复的场合（IM），**先不回，先想**。把"立刻"的压力还回去。
- 如果对方在群里 @ 你，**只回一句占位**："收到，我看一下再同步"——已经把球踢给对方"你等我"。
- 正式回复走 L3 以上渠道。让对方看到"我认真对待、而且要留痕"。
- 反之，对**我方有利的节点要尽快公开化**：拿到承诺就立刻 @ 相关方确认。

---

## 8. 使用本文件的方式（给 agent）

每次进入本 skill 时，按顺序答完以下清单再开始写三档方案：

```
[ ] 权力三角画出来了：Boss / 白手套 / 我 / 旁观者分别是谁？
[ ] 对方用的是哪个叙事框架？我要接还是换？
[ ] 当前对话在哪个书面化层级？目标推到哪一级？
[ ] 风险六项打分 → 档位初选
[ ] 用户真实诉求是哪一类？
[ ] 是否命中「常见局面速判表」里的某一行？
```

答完这张清单，三档方案就几乎是填空题了。
