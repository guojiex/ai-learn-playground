# Hook Patterns：钩子模式库

钩子（Hook）= 埋在回复里的话术陷阱。好钩子的共同特征：

1. **表面合理**，对方无法直接拒绝（拒绝就显得理亏）
2. **答 / 不答都有收益**：对方答了你拿到筹码，不答你拿到"不愿承诺"的把柄
3. **可公开化**：钩子句本身可以被大老板看到而不丢分

每个钩子给出：**埋点话术 / 预期反应 / 触发信号 / 反击三档**。

---

## 1. 书面化钩子（Written Commitment Hook）

把对方的口头话 / 立场逼到文档 / 群里。

### 埋点话术

> 为避免后续理解偏差，我把刚才对齐的原则（<对方说的话>）写进 <文档 / 群置顶 / 评论区>，后续按这个执行，你看 OK 吗？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **爽快同意** | 你拿到了书面承诺，后续可用 |
| **同意但拖延落地** | 你可以反复追 ping，显得对方不守承诺 |
| **拒绝 / 改口** | 证明他之前说的话不作数，白手套 / PUA 嫌疑坐实 |
| **装没看见** | 过 1-2 天 @ 一次，对方沉默即默认，你自己写 |

### 触发信号

- 对方说"不用写文档啦"、"口头对齐就行"、"别这么正式" → **强烈触发**（确认他怕留痕）
- 对方改口"我刚才不是这个意思" → **完全触发**（已经在逃避）

### 反击三档

**温和档：**
> 我理解。那我这边就按我之前的理解（<你本来的做法>）继续了，如果后续有误差再同步。

**标准档：**
> 不写的话 <具体风险，比如下游同学 / 新同事接手会没对齐>。我先写一版草稿同步给你，你改也行、改完对齐也行？

**强硬档（在群里 / 抄送老板）：**
> 为了让 <BigBoss / 产品 / 下游团队> 也能对齐，我还是把原则整理成一版写在 <群 / 文档>，@ 一下相关方。如果有异议请回复，无异议我按这个推进。

## 2. 成本量化钩子（Cost Quantification Hook）

对方用"没人力 / 成本高 / 很难"模糊化时，逼他给具体数字。

### 埋点话术

> 我这边也要给产品排期，方便拉齐一下几个数字吗：
> 1. 这个方案预计占用你们多少人日？
> 2. 对应哪位同学？什么时候能排进去？
> 3. 如果要这季度内完成，需要挤掉哪个优先级的活？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **给出具体数字** | 把事实 / 排期钉死，后续可追踪 |
| **"差不多就可以了" / 模糊回复** | 证明"没人力"是借口 |
| **反过来骂你"你们不讲道理"** | 情绪化 = 没理，可作为后续升级证据 |
| **把球踢给 leader** | 已把问题合规化升级，可继续推 |

### 触发信号

- 对方说"这个不好估" / "每个 case 不一样" → **触发**
- 对方给了数字但明显夸张（比如小改动要 10 人日）→ **触发**，可以追"这个数字能拆一下吗？具体是哪几块？"

### 反击三档

**温和档：**
> 了解。那我这边按我最初的方案先推了，有什么新的排期信息同步一下。

**标准档：**
> 那我整理一下我这边知道的工作量估算同步给你，我们对一下差异？如果确认是这么多人日，我会同步产品重新对齐优先级。

**强硬档：**
> 这个数字跟我这边了解到的差距较大。要不我们拉 <产品 / Boss> 一起对一下，把现状同步给他们，让他们决定优先级？

## 3. 责任归位钩子（Ownership Pinning Hook）

把对方模糊的"我们 / 系统 / 机器人 / 团队"逼到具体的人。

### 埋点话术

> 按你说的 <对方陈述，比如"机器人分拣要保证准确"> ，我想确认一下：
> 1. 当前 <这个流程 / 指标> 的 owner 是谁？
> 2. 最近一次的 <准确率 / 成功率> 数据是多少？
> 3. 如果出问题（比如分拣错了），后续找谁跟进合适？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **给出 owner 和数据** | 已经把责任人钉死；如果数据难看，对方话术立即失效 |
| **"这个大家都管"** | 典型推诿，"大家管 = 没人管"，可追"那我拉 <这些同学> 一起对一下？" |
| **"这个我们内部会看"** | 给对方自己出一个承诺 deadline："好，那 <日期> 前同步我们一下进展？" |

### 触发信号

- 对方用"我们" / "系统" / "流程"做主语但**说不出具体人名**→ **触发**
- 对方转述别人的话但**不愿意拉那个人来对** → **强烈触发**

### 反击三档

**温和档：**
> 我理解这边还在梳理。那我们先定个对齐时间，比如 <下周一> 大家一起看一下？

**标准档：**
> 那我们就按 <某时间> 前，由 <你推测的 owner 名字> 同步一下现状？我来约会。

**强硬档：**
> 既然 owner 不明确，这块我同步一下 Boss，让他帮我们拍一下归属，后面才好推进。

## 4. KPI 反射钩子（KPI Mirror Hook）

把对方推给你的活翻译成**对方自己 KPI / OKR 上的动作项**。

### 埋点话术

> 这个事如果这次推不动，对 <你们 OKR 条目 / 指标 X> 应该也有影响吧？要不要我帮你同步一下 <Boss / 产品>，一起看怎么合理排期？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **承认影响** | 你拿到对方的确认，后续推不动就是对方的 KPI 问题 |
| **"这个不算我们 OKR"** | 那就更不应该由我做，可顺势推回："那这个应该是谁的 OKR？我们拉他来看一下" |
| **紧急回到正题** | 证明 KPI 压力起效了，对方开始合作 |

### 触发信号

- 对方突然态度转好 / 主动说"我们来看一下" → **触发**（怕 KPI 掉）
- 对方说"OKR 的事你别管" → **触发**，记为"对 KPI 绑定反感" = 心虚

### 反击三档

**温和档：**
> 好，那你们这边有结果同步一下就行，我先不同步 Boss。

**标准档：**
> 行。为了后续对齐，我把这个 <影响点> 写进我们共同的项目文档里，大家都能看到进展。

**强硬档：**
> 既然你们不确定是不是 KPI 动作项，我索性周报里同步现状，让 Boss 帮我们拍一下优先级和归属。

---

## 5. 公开化钩子（Public Record Hook）

把私聊内容拽到群 / 文档 / 邮件里。

### 埋点话术

> 这块挺重要的，我在 <群 / 项目文档> 里同步一下刚才对齐的内容，顺便 @ 一下 <相关方> 让他们也知道进展，你看可以吗？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **同意** | 你合法地把私聊搬到公开场合，信息不对称消失 |
| **"不用吧这个我们私下对齐就行"** | 证明对方不想让更多人知道，**立即触发强钩** |
| **给理由不想公开**（比如"别让 XX 知道"）| 可以记录下来，后续作为"刻意隐藏信息"的证据 |

### 触发信号

- 对方反对公开化 → **强烈触发**
- 对方要求你"帮 TA 遮"、"别跟 XX 说" → **强烈触发**（这本身就是不正当动作）

### 反击三档

**温和档：**
> 好，那先不公开。我这边整理一份自用文档记录一下进展，后续有新情况再同步。

**标准档：**
> 这个议题 <下游 / 产品 / 老板> 早晚会问到，我倾向提前同步。如果你有顾虑我们先讨论一下再决定，行吗？

**强硬档：**
> 这个事涉及到 <下游团队 / 老板关注的指标>，我需要在 <文档 / 群> 里同步现状。你如果有不同意见，我们可以在文档里讨论。

---

## 6. 衡量标准钩子（Measurability Hook）

对方用模糊的"情绪大 / 不够 AI / 态度不好"PUA 时，逼他给出**可衡量的标准**。

### 埋点话术

> 我认真对待这个反馈。能不能明确一下：
> 1. 具体哪件事让你这么判断？
> 2. 你期望的动作是什么？（A 还是 B？）
> 3. 下次怎么算"做到了"？——有没有一个可观察的信号或指标？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **给出具体标准** | PUA 武器废掉，你按标准做即可 |
| **"你自己体会" / 继续模糊** | 证明是 PUA，没有实际内容，可礼貌无视 |
| **恼羞成怒** | 情绪化 = 没理，作为后续证据 |

### 触发信号

- 对方不愿意给具体例子 → **触发**（PUA 坐实）
- 对方绕回"就是你态度问题" → **触发**（人身评价无法被证伪 = 无效指控）

### 反击三档

**温和档（私聊）：**
> 了解。那我先按 <我自己的理解> 做，如果有具体 case 需要调整你告诉我。

**标准档（书面化 1on1 纪要）：**
> 上次 1on1 的反馈我这边整理了一下（附文档）：第一部分是具体动作，第二部分是我还没完全 get 的，我们下次再对齐一下？

**强硬档（向 HR / Skip 准备证据时）：**
> 多次希望你给具体 case 但都没得到。为了避免后续误会，我把之前我做的几个项目 summary 整理一下，你有具体意见时一起看。

## 7. 优先级确认钩子（Priority Pinning Hook）

对方塞活时，逼他排优先级。

### 埋点话术

> 这个我可以接。我这边目前有 A / B / C 在做，排期到 <日期>。如果这件事要这周做完，需要把 A / B / C 里哪个降优先级？你拍一下？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **明确降优先级** | 你得到书面授权，出问题是对方的锅 |
| **"你自己排一下"** | 典型推卸决策，回："我初步排是 <给个顺序>，你看可以吗？"逼他签字 |
| **"都要做"** | 回："那我这边人力跟不上，是不是需要申请加人 / 加班资源？" |
| **改口说"不用这么急"** | 你成功把"紧急" PUA 消解 |

### 触发信号

- 对方不愿意拍优先级 → **触发**
- 对方说"都要做" → **触发**（资源冲突，可升级）

### 反击三档

**温和档：**
> 好那我按 <我初步的顺序> 推进，有新情况同步。

**标准档：**
> 我把当前排期整理一份发到群 / 发给你，你确认一下没问题我就按这个推。

**强硬档：**
> 这几个都是这季度 OKR 相关，如果都要做，我申请一下 <加人 / 延期 / 加班资源>，我同步给 Boss 决策一下？

---

## 8. 时间戳钩子（Timestamp Hook）

让对方给 deadline，或让对方的沉默变成默认同意。

### 埋点话术（两种）

**A 要对方承诺时间：**
> 这块的 <具体动作> 你们大概 <什么时间> 能出 <具体交付物>？方便的话我按这个时间点同步给 <产品 / 下游>。

**B 让沉默等于默认：**
> 以上是我对 <议题> 的整理和下一步动作。如果 <日期> 前没有异议，我按这个推进。

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **给时间** | 拿到 deadline，可追踪 |
| **"不好说"** | 可以推："那我们先定一个保守的时间点作为初始 baseline？<具体日期>" |
| **沉默** | B 方案下你自动推进，合规 |
| **到期反悔** | 你有截图，反悔成本很高 |

### 触发信号

- 对方迟迟不给 deadline → **触发**（怕被钉死）
- 对方到期反悔但没说具体新时间 → **触发**，追问"具体新 deadline 是？"

### 反击三档

**温和档：**
> 那我们先按 <baseline 日期> 做参考，到时再看情况调整。

**标准档：**
> 这个 deadline 我记在文档里了（附链接），到时我们一起 review 下。

**强硬档：**
> 已经 <多久> 没有 deadline，我这边没法给下游一个说法。要不我们拉 Boss 对一下优先级？

## 9. 文档引用钩子（Document Citation Hook）

把对方话架在已有文档 / 流程 / 规定上。

### 埋点话术

> 我理解你的想法。不过按我们现有的 <文档 / RACI / 流程规范>（链接：xxx），这块应该是 <正确做法>。你希望我们 <特殊处理> 的话，需要先把这个规范调整一下吗？

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **承认按文档来** | 你赢，回到规则下 |
| **"文档过时了"** | 回："那更新一下？我可以起草" —— 你拿到了修文档的主动权 |
| **"别太教条"** | 可以记录下来，后续出问题对方无法甩锅给"没按流程" |

### 触发信号

- 对方反对按文档 → **触发**
- 对方要求你"灵活处理" → **触发**（默认规则对他不利）

### 反击三档

**温和档：**
> 那我这次先 <特殊处理>，但我们是不是要同步更新一下文档，避免后续反复？

**标准档：**
> 为了后续统一，我起草一版新的 <文档 / 流程> 供大家 review，这次的 case 就按新版的来？

**强硬档：**
> 既然要偏离现有规范，这个决策应该拉 Boss 一起看，我约一下会？

## 10. 互惠陷阱（Reciprocity Trap）

用**小让步**换**大承诺**，或者**接受对方的小要求**让他难以拒绝你的大要求。

### 埋点话术

> 这次 <对方的诉求> 我这边可以配合下，<具体让步>。不过相应的，能不能麻烦你们也 <我的诉求>？这样我们这边推起来也顺。

### 预期反应

| 对方反应 | 暴露了什么 |
|---------|-----------|
| **同意互换** | 合规交易，书面化即可 |
| **只要你让步不愿回报** | 对方吃独食姿态暴露，可以升级："这样单向让我这边不太好给产品交代……" |
| **否认自己有诉求** | 回撤你的让步："那我按原来的计划推了" |

### 触发信号

- 对方拿了好处不肯回报 → **强烈触发**（可以公开化）

### 反击三档

**温和档：**
> 那这次我还是按我原来的计划推，等后面合适的节点再配合你们。

**标准档：**
> 为了后续对齐，我把我这边的让步和预期的对等动作写进 <协作文档>，大家都心里有数。

**强硬档（对方反悔时）：**
> 我上次的让步是基于你答应 <XX>。既然 <XX> 没做，我需要把我的那部分也撤回。

## 钩子使用原则

1. **一次最多埋 2-3 个钩子**，多了对方会起疑心
2. **温和档只埋 1 个钩子**，强硬档可埋 3 个
3. **钩子措辞必须中性、合理**，能被截图发给大老板而不丢分
4. **埋完后记录下来**（agent 在每次输出时必须列钩子清单），反击模式要用
5. **钩子 > 表态**：任何一轮对话里，**埋钩子比表达立场更重要**；立场可以缓、钩子必须埋

## 反击模式：钩子回收流程（给 agent）

当用户贴来对方的回复时，agent 按以下流程走：

```
1. 从上一轮对话的「钩子清单」中逐条核对：
   - 哪些钩子对方直接踩中？（按触发信号判断）
   - 哪些钩子对方绕开？（绕开也是一种信号）
   - 有没有对方预料之外的新招？

2. 输出「钩子回收表」：
   | 钩子 | 是否触发 | 对方原话 | 可利用强度 |

3. 针对踩中的钩子，从本文件对应的「反击三档」中选招，组合成本轮反击三档

4. 新一轮仍然继续埋钩子（战斗不结束，钩子不断）
```
